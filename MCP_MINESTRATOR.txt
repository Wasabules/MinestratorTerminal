# Connecter votre IA à MineStrator (serveur MCP)

---

## 1. C'est quoi ?

MineStrator expose un **serveur MCP** (Model Context Protocol) : le standard ouvert qui permet
aux assistants IA (Claude Code, Cursor, et tout client compatible MCP) de **gérer vos serveurs
de jeu à votre place**, avec votre autorisation.

Concrètement, une fois connectée, votre IA peut :

- **Diagnostiquer** : état temps réel, logs console, événements (crashs, OOM, disque plein),
  historique de consommation CPU/RAM — « pourquoi mon serveur lag depuis hier ? »
- **Gérer les fichiers** : lire/éditer les configs (`server.properties`, YAML de plugins…),
  chercher, déplacer, archiver (.zip), extraire.
- **Agir** : démarrer/arrêter/redémarrer, commandes console, modération de joueurs
  (kick/ban/whitelist/op).
- **Installer du contenu** : versions Minecraft (Vanilla/Paper/Forge/NeoForge/Fabric…),
  plugins, mods, modpacks, mondes, datapacks — avec sélection de versions **compatibles**
  (catalogues Spigot, Modrinth, CurseForge), mods Hytale, mods Steam Workshop.
- **Sauvegarder / restaurer** : snapshots à la demande, smartbackups quotidiens, suivi des jobs.
- **Administrer la MyBox** : créer/supprimer/redimensionner des serveurs, activer/désactiver,
  adresses en `.minesr.com` / `.mine.fun`, ports supplémentaires, tâches planifiées (cron),
  bases de données (MyBox classiques).

60 outils, 9 familles. L'IA reçoit pour chaque outil un mode d'emploi précis (quand l'utiliser,
quoi vérifier avant, ce qui est destructif) — en français.

**URL du serveur** : `https://mcp.sttr.io/minestrator`

## 2. Prérequis : votre clé API

1. Panel MineStrator → **Compte → Clés API** → créer une clé.
2. La clé affichée par le panel est **directement au bon format** : copiez-la telle quelle
   dans l'en-tête `Authorization: Bearer` — **aucun encodage à faire**.

⚠️ Ne transformez pas la clé (pas de ré-encodage base64 ou autre) : elle est déjà prête à
l'emploi — une clé ré-encodée est refusée (« clé invalide »).

⚠️ **Cette clé donne accès à votre compte** (mêmes droits que vous sur vos serveurs, ou que les
permissions du sous-compte qui l'a créée). Traitez-la comme un mot de passe. Elle est révocable
à tout moment depuis le panel.

## 3. Connexion par client

> Conseil transverse : collez la **valeur littérale** de la clé, telle qu'affichée par le
> panel, dans les commandes/configs. Toute interpolation shell du type `$(...)` à l'intérieur
> d'une commande d'ajout peut produire silencieusement un en-tête vide → erreur « clé invalide »
> incompréhensible (dans ce cas : supprimer le serveur MCP et le ré-ajouter proprement).

### Claude Code (CLI)

```bash
claude mcp add --transport http minestrator https://mcp.sttr.io/minestrator \
  --header "Authorization: Bearer VOTRE_CLE_API"
```

Puis dans une session : `/mcp` pour vérifier la connexion. Les outils apparaissent sous
`mcp__minestrator__*`.

### Cursor

Fichier `~/.cursor/mcp.json` (global) ou `.cursor/mcp.json` (projet) :

```json
{
  "mcpServers": {
    "minestrator": {
      "url": "https://mcp.sttr.io/minestrator",
      "headers": { "Authorization": "Bearer VOTRE_CLE_API" }
    }
  }
}
```

### Autres clients MCP (transport HTTP « streamable »)

Tout client supportant le transport **Streamable HTTP** avec en-têtes personnalisés fonctionne :
URL `https://mcp.sttr.io/minestrator` + en-tête `Authorization: Bearer <clé API>`.
Pour les clients limités à stdio, le pont `mcp-remote` fonctionne (usage avancé, non supporté
officiellement) :

```json
{
  "mcpServers": {
    "minestrator": {
      "command": "npx",
      "args": ["mcp-remote", "https://mcp.sttr.io/minestrator",
               "--header", "Authorization: Bearer VOTRE_CLE_API"]
    }
  }
}
```

### Connecteurs claude.ai (web) / intégrations OAuth

Pas encore disponibles — les connecteurs web exigent OAuth 2.1, prévu dans une prochaine
itération. En attendant : Claude Code, Cursor, ou tout client acceptant un en-tête personnalisé.

## 4. Limiter ce que l'IA peut faire (toolsets & mode lecture seule)

### Mode lecture seule

Pour donner à une IA un accès **observation uniquement** (diagnostic, monitoring, lecture de
fichiers et de logs — aucune modification possible), ajoutez `?readonly=1` à l'URL :

```
https://mcp.sttr.io/minestrator?readonly=1
```

Seuls les 24 outils de lecture sont alors exposés. Les outils modifiants (power, écriture de
fichiers, installations, suppressions…) ne sont pas simplement masqués : ils n'existent pas
pour cette connexion, même appelés directement. Cumulable avec `?toolsets=`
(ex. `?toolsets=core,files&readonly=1`).

### Toolsets

Par défaut, les 60 outils sont exposés. Pour restreindre par domaine, ajoutez `?toolsets=` à l'URL :

```
https://mcp.sttr.io/minestrator?toolsets=core,files
```

| Toolset | Contenu |
|---|---|
| `core` | compte, liste des serveurs, diagnostic, monitoring, maintenances infra |
| `files` | fichiers (lecture/écriture/recherche/archives/déplacements) |
| `actions` | power, commandes console, modération joueurs |
| `content` | versions, plugins, mods, modpacks, mondes, datapacks (recherche + install) |
| `backups` | snapshots, smartbackups, jobs de sauvegarde/restauration |
| `mybox` | fiche MyBox, création/suppression/resize/enable de serveurs |
| `schedules` | tâches planifiées (cron) |
| `databases` | bases de données MySQL/MariaDB (MyBox classiques) |
| `config` | adresse du serveur, démarrage (jar/params), Java/Docker, ports |

Exemple lecture seule stricte : `?readonly=1`. Exemple sans actions destructives de
box : tout sauf `mybox`. Un toolset ou une valeur `readonly` inconnus → erreur 400 explicite.

## 6. Sécurité — ce qu'il faut savoir

- **Vos permissions s'appliquent toujours** : le serveur MCP transmet votre clé à l'API
  MineStrator telle quelle — un sous-compte n'a que ses permissions, le rate-limiting et la
  journalisation d'activité (avec votre IP réelle) restent actifs. Le serveur MCP ne stocke
  rien : aucune session, aucune clé conservée.
- **Garde-fous intégrés** : les opérations destructives (suppression de serveur, restauration,
  reset de mot de passe DB…) demandent à l'IA de reformuler la conséquence et d'obtenir votre
  confirmation explicite ; la suppression d'un serveur impose un snapshot de sécurité préalable
  (2 étapes) ; les refus volontaires sont explicites (« ⛔ Refus : … »).
- **Anti-injection** : tout contenu provenant du jeu (console, fichiers, noms saisis par des
  joueurs) est balisé « données non fiables » — l'IA a pour consigne de ne jamais l'interpréter
  comme des instructions.
- **À savoir** : l'IA peut lire vos **accès complets de bases de données et SFTP** (c'est voulu
  — nécessaire pour configurer un plugin, par exemple). Si vous ne le souhaitez pas, excluez
  les toolsets `databases`/`core` ou utilisez une clé d'un sous-compte restreint.
- En cas de doute, **révoquez la clé** depuis le panel : effet immédiat (≤ 60 s).

## 8. FAQ / dépannage

| Symptôme | Cause probable | Solution |
|---|---|---|
| « Clé API invalide ou révoquée » | Clé tronquée/modifiée à la copie, ré-encodée à tort, ou en-tête vide (interpolation shell) | Recopier la clé exactement telle qu'affichée sur le panel, supprimer puis ré-ajouter le serveur MCP |
| 401 dès la connexion | En-tête `Authorization` absent | Vérifier la config du client (headers) |
| « Limite de requêtes atteinte » | Rate-limit API (fenêtre 60 s, par endpoint) | L'IA doit attendre ~60 s ; éviter les boucles de polling |
| Un nouvel outil n'apparaît pas | Catalogue chargé à la connexion | Reconnecter le serveur MCP (`/mcp` dans Claude Code) ou redémarrer la session |
| « Votre offre n'inclut pas de base de données » | Offres Pro : pas de DB incluse (choix produit) | Les bases de données sont disponibles sur les MyBox classiques |
| L'IA refuse d'écrire un `.jar` | Édition réservée aux fichiers texte | Installer les binaires via les outils d'installation (mods/plugins) ou par SFTP |

Le service est **inclus** dans votre offre MineStrator — les limites usuelles de l'API
s'appliquent.
